Parameters:

zThe output vector of the routine. z could be y, but not y + k (i.e. an overlapping area over y, but starting from a base index different from y).
yThe y input vector
alphaThe alpha scalar
cMThe ELL non zero values allocation pointer
rPThe ELL column indices allocation pointer

rSthe array containing the row sized (in non zero elements)
rowsthe rows count
xthe x vector

Computes double precision z = alpha*A*x + beta*y, with A stored in Hacked ELLpack Format on GPU.

Parameters:
handleThe spgpu handle used to call this routine
zThe output vector of the routine. z could be y, but not y + k (i.e. an overlapping area over y, but starting from a base index different from y).
yThe y input vector
alphaThe alpha scalar
cMThe HELL non zero values allocation pointer
rPThe HELL column indices allocation pointer
hackSizeThe constant size of every hack (must be a multiple of 32).
hackOffsetsthe array of base index offset for every hack of HELL non zero values allocation and HELL indices allocation.
rSthe array containing the row sized (in non zero elements)
rIdx(optional) An array containing the row index per every row (i.e. the reorder array) of the Hell matrix. Pass NULL if you don't use a reorder array (i.e. the k-th row is stored in the k-th position in the HELL format).
rowsthe rows count
xthe x vector
betathe beta scalar
baseIndexthe ELL format base index used (i.e. 0 for C, 1 for Fortran).
